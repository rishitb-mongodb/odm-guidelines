# Django MongoDB Backend

**Language / Ecosystem:** Python / Django
**Repository:** [mongodb/django-mongodb-backend](https://github.com/mongodb/django-mongodb-backend)

Django MongoDB Backend is the official MongoDB database backend for Django. It uses PyMongo to connect to MongoDB and plugs into Django's ORM, enabling standard Django models, QuerySets, migrations, and contrib apps (Admin, Auth, GeoDjango) to work against MongoDB. The current release targets Django 6.0 and Python 3.10+.

## Guidelines Triage

| Feature | Triage Status |
|---------|---------------|
| [Query Builder](../guidelines/query-builder/query-builder.md) | ![Done](https://img.shields.io/badge/Done-brightgreen?style=flat) — Standard Django QuerySet API (`filter()`, `exclude()`, `order_by()`, `get()`, `first()`); Q objects with `\|`, `&`, `~`, `^`; array lookups (`contains`, `contained_by`, `overlap`, `len`); embedded model traversal via double-underscore (`awards__wins__gt=150`); index querying and slice querying on `EmbeddedModelArrayField`; aggregation pipeline via `raw_aggregate()` (requires `MongoManager` on the model); `raw()`, `extra()`, `prefetch_related()`, and `select_for_update()` are not supported |
| [Bulk Operations](../guidelines/bulk-operations/bulk-operations.md) | ![Backlog](https://img.shields.io/badge/Backlog-lightgrey?style=flat) — Django's `bulk_create()` works; ordered/unordered `BulkWrite` requires the PyMongo escape hatch via `connections["default"].database.client` |
| [Relationship Mapping](../guidelines/relationship-mapping/relationship-mapping.md) | ![Done](https://img.shields.io/badge/Done-brightgreen?style=flat) — `EmbeddedModelField` and `EmbeddedModelArrayField` (recommended, no `$lookup`); `ForeignKey`, `OneToOneField`, and `ManyToManyField` supported across collections but use slow `$lookup`; relational fields cannot be used inside `EmbeddedModel` classes or as the `base_field` of an `ArrayField`; prefer denormalized embedded patterns for performance |
| [Index Management](../guidelines/index-management/index-management.md) | ![Done](https://img.shields.io/badge/Done-brightgreen?style=flat) — Single-field, compound, multikey (automatic for array fields), partial, unique, geospatial, and embedded document indexes via `Meta.indexes`; `EmbeddedFieldIndex` for subfield indexes within embedded models (v6.0.2+); `SearchIndex` and `VectorSearchIndex` classes for Atlas indexes; embedded/array field index updates cannot be tracked by migrations |
| [Transactions](../guidelines/transactions/transactions.md) | ![Done](https://img.shields.io/badge/Done-brightgreen?style=flat) — Custom `from django_mongodb_backend import transaction` module (Django's native `django.db.transaction` is not supported); `@transaction.atomic` decorator and `with transaction.atomic():` context manager; `transaction.on_commit()` for post-commit callbacks; requires replica set or sharded cluster; no savepoints, no `union()` inside transactions, DDL transactions unsupported |
| [BSON Data Type Support](../guidelines/bson-data-types/bson-data-types.md) | ![Done](https://img.shields.io/badge/Done-brightgreen?style=flat) — Standard Django field types map to MongoDB BSON types; `ObjectIdField` from `django_mongodb_backend.fields` replaces `AutoField`/`BigAutoField`/`SmallAutoField` (auto-fields are not supported); `DateTimeField` has no microsecond granularity; `DurationField` stores milliseconds not microseconds; `JSONField` partially supported (cannot distinguish JSON null from SQL null, some Q object queries are problematic); `CompositePrimaryKey` and `GeneratedField` are not supported |
| [Array Fields](../guidelines/array-fields/array-fields.md) | ![Done](https://img.shields.io/badge/Done-brightgreen?style=flat) — `ArrayField` (scalar values) and `EmbeddedModelArrayField` (embedded model instances) from `django_mongodb_backend.fields`; lookups: `contains`, `contained_by`, `overlap`, `len`; embedded model arrays support index querying (`cast__0__last_name`) and slice querying (`cast__0_2__last_name`); supported lookups on embedded array elements: `exact`, `iexact`, `in`, `gt`, `gte`, `lt`, `lte` |
| [Embedded Fields](../guidelines/embedded-fields/embedded-fields.md) | ![Done](https://img.shields.io/badge/Done-brightgreen?style=flat) — `EmbeddedModelField` from `django_mongodb_backend.fields` with `EmbeddedModel` base class; double-underscore dot notation for nested field queries and updates; `EmbeddedFieldIndex` for indexing subfields (v6.0.2+); relational fields (`ForeignKey`, etc.) cannot be used inside `EmbeddedModel` |
| [Polymorphic Array/Embedded Fields](../guidelines/polymorphic-fields/polymorphic-fields.md) | ![Done](https://img.shields.io/badge/Done-brightgreen?style=flat) — Polymorphic embedded models supported; shared fields and type-specific fields can both be queried; if multiple embedded model types declare the same field name with different data types, Django uses the first matching field definition |
| [Security Features (CSFLE & QE)](../guidelines/security-features/security-features.md) | ![Done](https://img.shields.io/badge/Done-brightgreen?style=flat) — Queryable Encryption (QE) is supported and documented as a first-class feature; Client-Side Field Level Encryption (CSFLE) is not supported |
| [Joins ($lookup)](../guidelines/joins/joins.md) | ![Done](https://img.shields.io/badge/Done-brightgreen?style=flat) — `ForeignKey`, `OneToOneField`, and `ManyToManyField` generate `$lookup` implicitly; custom `$lookup` pipelines via `raw_aggregate()`; `$lookup` is flagged as slow in the compatibility docs — embedded models are the recommended alternative |
| [Atlas Search](../guidelines/atlas-search/atlas-search.md) | ![Done](https://img.shields.io/badge/Done-brightgreen?style=flat) — Dedicated expression classes in `django_mongodb_backend.expressions`: `SearchText`, `SearchPhrase`, `SearchEquals`, `SearchAutocomplete`, `SearchExists`, `SearchIn`, `SearchRange`, `SearchRegex`, `SearchWildcard`, `SearchGeoShape`, `SearchGeoWithin`, `SearchMoreLikeThis`; compose with `CompoundExpression` (`must`, `should`, `must_not`, `filter`, `minimum_should_match`) or bitwise `\|`/`&`/`~` operators; relevance scoring via `SearchScoreOption`; all used through `annotate()` on a QuerySet; `SearchIndex` class for index management; some advanced `SearchIndex` field types (`numPartitions`, `analyzers`, `storedSource`, `synonyms`, `typeSets`) are not yet supported |
| [Vector Search](../guidelines/vector-search/vector-search.md) | ![Done](https://img.shields.io/badge/Done-brightgreen?style=flat) — `SearchVector` expression from `django_mongodb_backend.expressions`; parameters: `path`, `query_vector`, `limit`, `num_candidates`, `exact` (ENN), `filter`; used via `annotate()`; ANN (default) and ENN (`exact=True`) both supported; `VectorSearchIndex` class for index management; only basic vector index configuration supported (advanced index fields not yet available) |
| [Logging](../guidelines/logging/logging.md) | ![Done](https://img.shields.io/badge/Done-brightgreen?style=flat) — Standard Django `logging` framework applies; PyMongo `monitoring.CommandListener` available via the escape hatch (`connections["default"].database.client`) for low-level command logging; no built-in slow query detection; Django Debug Toolbar is partially supported |
| [Escape Hatch](../guidelines/escape-hatch/escape-hatch.md) | ![Done](https://img.shields.io/badge/Done-brightgreen?style=flat) — `connections["<key>"].database.client` returns the underlying PyMongo `MongoClient` for full driver access; `raw_aggregate()` on a model with `MongoManager` for direct aggregation pipeline execution; results from `raw_aggregate()` are deferred model instances (omitted fields can be loaded on demand) |
| [Performance Benchmarks](../guidelines/performance-benchmarks/performance-benchmarks.md) | ![Backlog](https://img.shields.io/badge/Backlog-lightgrey?style=flat) |
