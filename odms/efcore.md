# EF Core (MongoDB Entity Framework Core Provider)

**Language / Ecosystem:** C# / .NET
**Repository:** [mongodb/mongo-efcore-provider](https://github.com/mongodb/mongo-efcore-provider)

The MongoDB EF Core Provider is a MongoDB-maintained database provider for Microsoft's Entity Framework Core. It allows .NET developers to use EF Core's DbContext, LINQ queries, and change tracking with MongoDB as the backing store, sitting on top of the official MongoDB C# driver.

## Guidelines Triage

| Feature | Triage Status |
|---------|---------------|
| [Query Builder](../guidelines/query-builder/query-builder.md) | ![Done](https://img.shields.io/badge/Done-brightgreen?style=flat) — Full LINQ translation to MongoDB Query API; `Mql.Exists()`, `Mql.IsMissing()`, `Mql.IsNullOrMissing()` helpers; some advanced client-side projections planned for future versions |
| [Bulk Operations](../guidelines/bulk-operations/bulk-operations.md) | ![Backlog](https://img.shields.io/badge/Backlog-lightgrey?style=flat) — `AddRange()`/`RemoveRange()` via `SaveChanges()` only; set-based `ExecuteUpdate` and `ExecuteDelete` (EF Core 7+) not yet supported; no ordered/unordered mode or per-record result reporting |
| [Relationship Mapping](../guidelines/relationship-mapping/relationship-mapping.md) | ![Done](https://img.shields.io/badge/Done-brightgreen?style=flat) — `OwnsOne()`/`OwnsMany()` for embedding; lazy loading not supported; complex `Include().ThenInclude()` chains have limitations |
| [Index Management](../guidelines/index-management/index-management.md) | ![Done](https://img.shields.io/badge/Done-brightgreen?style=flat) — `HasIndex()` in `OnModelCreating()`; unique, sparse, compound, descending indexes; `HasCreateIndexOptions()` for MongoDB-specific options; indexes created only on `Database.EnsureCreated()`; geospatial indexes not supported |
| [Transactions](../guidelines/transactions/transactions.md) | ![Done](https://img.shields.io/badge/Done-brightgreen?style=flat) — `SaveChanges()` transactional by default since v8.1; explicit transactions via `MongoTransactionManager` (v9.0.3+); requires replica set |
| [BSON Data Type Support](../guidelines/bson-data-types/bson-data-types.md) | ![Done](https://img.shields.io/badge/Done-brightgreen?style=flat) — ObjectId, Decimal128, all CLR numeric types; DateTime defaults to UTC (`[BsonDateTimeOptions]` for local); GUID defaults to Standard (UUID subtype 4); `Mql.IsNullOrMissing()` for null vs missing distinction |
| [Array Fields](../guidelines/array-fields/array-fields.md) | ![Done](https://img.shields.io/badge/Done-brightgreen?style=flat) — `string[]`, `List<T>`, `ICollection<T>`, `Dictionary<K,V>` field types; basic LINQ array operators; advanced update operators (`$push`, `$arrayFilters`) require driver-level access |
| [Embedded Fields](../guidelines/embedded-fields/embedded-fields.md) | ![Done](https://img.shields.io/badge/Done-brightgreen?style=flat) — `OwnsOne()`/`OwnsMany()` for nested documents; LINQ dot notation; partial updates not supported — whole-document updates only; discriminators on sub-documents not supported (EF Core has no concept of this) |
| [Polymorphic Array/Embedded Fields](../guidelines/polymorphic-fields/polymorphic-fields.md) | ![Done](https://img.shields.io/badge/Done-brightgreen?style=flat) — Table-Per-Hierarchy (TPH) via `_t` discriminator (default from v10.0); `HasDiscriminator()` fluent API; discriminators on embedded (non-root) documents not supported |
| [Security Features (CSFLE & QE)](../guidelines/security-features/security-features.md) | ![Done](https://img.shields.io/badge/Done-brightgreen?style=flat) — QE equality (`IsEncryptedForEquality()`) and range (`IsEncryptedForRange()`, v9.0.1+) supported; substring queries not yet available; requires `MongoDB.Driver.Encryption` package and `crypt_shared` library; MongoDB M10+ only |
| [Joins ($lookup)](../guidelines/joins/joins.md) | ![Backlog](https://img.shields.io/badge/Backlog-lightgrey?style=flat) — Basic `Include()` for single navigation properties works; complex `Include().ThenInclude()` chains and `$lookup` sub-pipelines not yet supported; planned for future releases |
| [Atlas Search](../guidelines/atlas-search/atlas-search.md) | ![Won't Do](https://img.shields.io/badge/Won't_Do-red?style=flat) — Not supported; no EF Core equivalent for `$search`; use raw driver aggregation with `$search` stage via `MongoClient` escape hatch instead |
| [Vector Search](../guidelines/vector-search/vector-search.md) | ![Done](https://img.shields.io/badge/Done-brightgreen?style=flat) — Full LINQ `VectorSearch()` integration since v8.3.3/v9.0.3; `DotProduct`, `Cosine`, `Euclidean` metrics; `.Prefilter()` and `__score` metadata; Atlas only |
| [Logging](../guidelines/logging/logging.md) | ![Done](https://img.shields.io/badge/Done-brightgreen?style=flat) — Standard EF Core `ILogger` / `DbLoggerCategory.Database.Command` integration; query compilation events (v10.0+); no built-in slow query threshold (use `DbCommandInterceptor` for custom detection) |
| [Escape Hatch](../guidelines/escape-hatch/escape-hatch.md) | ![Done](https://img.shields.io/badge/Done-brightgreen?style=flat) — Full driver access via `MongoClient` from DbContext setup; `GetDatabase()`/`GetCollection<T>()` for raw operations; session and transaction passthrough |
